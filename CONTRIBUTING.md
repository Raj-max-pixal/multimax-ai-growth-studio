# 🚀 Getting Started

### 1️⃣ Fork the Repository

Click the **Fork** button at the top-right of the GitHub page.

---

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/multimax-ai-growth-studio.git
```

---

### 3️⃣ Navigate to the Project

```bash
cd multimax-ai-growth-studio
```

---

### 4️⃣ Install Dependencies

```bash
npm install
```

or

```bash
pnpm install
```

---

### 5️⃣ Start Development

```bash
npm run dev
```

---

# 🌿 Branch Naming Convention

Create a new branch before making changes.

| Type          | Example              |
| ------------- | -------------------- |
| Feature       | feature/ai-dashboard |
| Bug Fix       | fix/login-error      |
| Documentation | docs/readme-update   |
| Refactor      | refactor/sidebar     |
| Performance   | perf/api-cache       |

Example:

```bash
git checkout -b feature/new-dashboard
```

---

# 💻 Coding Guidelines

Please follow these standards:

* Write clean and readable code.
* Use meaningful variable names.
* Keep components modular.
* Follow project formatting rules.
* Add comments only when necessary.
* Test before submitting.

---

# 📝 Commit Message Format

Use descriptive commit messages.

Examples:

```text
feat: add AI campaign generator

fix: resolve authentication bug

docs: update installation guide

style: improve dashboard layout

refactor: optimize API service
```

---

# 🐛 Reporting Bugs

Before creating a bug report:

* Search existing issues.
* Reproduce the problem.
* Include screenshots if applicable.
* Describe expected vs actual behavior.
* Mention your operating system and browser.

---

# 💡 Suggesting Features

Feature requests should include:

* Problem statement
* Proposed solution
* Benefits
* Optional mockups or examples

---

# 🔀 Pull Request Checklist

Before submitting your PR:

* ✅ Code builds successfully
* ✅ No linting errors
* ✅ Tested locally
* ✅ Documentation updated
* ✅ Commit messages are meaningful
* ✅ Branch is up to date

---
